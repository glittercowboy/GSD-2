**C5 — tauri.conf.json: Restrict assetScope**

Replace the `assetScope` block:
```json
"assetScope": {
  "allow": ["$RESOURCE/**", "$APP/**"],
  "deny": []
}
```

`$RESOURCE` = the app's bundled resources directory (where `dep_screen.html` lives).
`$APP` = the app's data directory. This prevents `asset://` from reading arbitrary files.

**H1 — Create capabilities/main.json**

Create the directory `packages/mission-control/src-tauri/capabilities/` and create `main.json`:

```json
{
  "$schema": "https://schema.tauri.app/config/2/capabilities",
  "identifier": "main-capability",
  "description": "Capability for the main application window",
  "windows": ["main"],
  "permissions": [
    "core:default",
    "core:window:default",
    "core:window:allow-close",
    "core:window:allow-set-size",
    "core:window:allow-set-position",
    "core:window:allow-set-title",
    "core:window:allow-create",
    "dialog:default",
    "dialog:allow-open",
    "dialog:allow-save",
    "opener:default",
    "opener:allow-open-url",
    "updater:default",
    "updater:allow-check",
    "updater:allow-download-and-install",
    "deep-link:default",
    "window-state:default",
    {
      "identifier": "core:event:default",
      "allow": [{ "event": "oauth-callback" }]
    }
  ]
}
```

This explicitly grants only the permissions the app needs. The IPC commands defined in `invoke_handler` (like `get_credential`, `set_credential`, etc.) are custom Tauri commands and are automatically available to the window that calls them — the capability file controls plugin permissions.

    if let tauri::WindowEvent::Destroyed = event {

            // M12: Only kill Bun when no windows remain (multi-window support)
